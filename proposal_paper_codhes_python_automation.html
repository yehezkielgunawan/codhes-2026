From Human-Centric to Machine-Optimized: A Technolinguistic Analysis of Documentation Readability in llm.txt Standards

Proposal & Alur Kerja Otomatisasi Komputasional Berbasis Ekosistem Python Modern
Profil Peneliti: Lulusan Magister Sistem Informasi (M.SI) & Software Engineer
Arsitektur Stack: Python CLI via `uv` Package Manager & `Crawl4AI` Framework
Metodologi Akademik: Kuantitatif Komputasional (Automated Technolinguistic Compliance Auditor)

1. Justifikasi Pemilihan Ekosistem Python & Alat `uv`

Peralihan dari arsitektur JavaScript ke ekosistem Python merupakan keputusan strategis untuk memperkuat validitas metodologis di lingkungan akademik (khususnya Digital Humanities). Python menyediakan pustaka asli (induk) yang menjadi acuan standar industri dalam penambangan teks (text mining) dan pemrosesan bahasa alami (NLP).

Proyek ini dikelola menggunakan uv (tool manajemen paket Python dari Astral). Pilihan ini menawarkan beberapa keuntungan teknis operasional:

  • Kecepatan Eksosferik: Proses instalasi pustaka dan resolusi dependensi berjalan jauh lebih cepat dibandingkan pip konvensional karena ditulis dalam bahasa Rust.
  • Isolasi Environment yang Bersih: Menjamin replikabilitas kode di komputer penguji atau penelaah konferensi tanpa merusak konfigurasi sistem global.
  • Eksekusi CLI Instan: Mempermudah automasi pengujian menggunakan perintah CLI yang ringkas untuk kebutuhan batch processing.

2. Logika Pipeline Data: Input Berkas Teks & Deteksi Otomatis

Untuk menggantikan proses manual, sistem dirancang sebagai sebuah Automated Compliance Auditor dengan alur kerja berbasis berkas masukan (input) teks:

  1. Membaca Input Berkas: Sistem membaca file urls.txt yang berisi daftar base URL domain teknologi target secara baris demi baris.
  2. Pemeriksaan Awal (Pre-flight Check): Skrip melakukan HTTP GET request otomatis menggunakan pustaka requests menuju folder root domain untuk mendeteksi keberadaan standar berkas LLM: /llm.txt atau /llms.txt.
  3. Penyaringan Korpus (Screening Mechanism): Jika berkas respons mengembalikan HTTP Status 200 OK, platform dinyatakan lolos audit awal dan datanya langsung diunduh sebagai plain text. Jika tidak ditemukan, sistem otomatis melompati (skip) domain tersebut.
  4. Deep Scraping via Crawl4AI: Untuk domain yang lolos, framework Crawl4AI diaktifkan secara asinkron untuk menyisir dan mengekstrak seluruh halaman dokumentasi manusia, lalu mengubahnya menjadi format Markdown bersih tanpa tag sampah (navigasi, sidebar, iklan).
Penguatan Metodologi pada Paper:
Mekanisme penapisan otomatis ini menaikkan bobot akademik riset pada Bab Metodologi. Peneliti dapat mengklaim penggunaan teknik Purposive Sampling Terotomatisasi, di mana data final disaring secara objektif oleh mesin berdasarkan kepatuhan kepemilikan standar berkas mesin (LLM text compliance).

3. Arsitektur Kode Otomatisasi (Python Pipeline Script)

Berikut adalah implementasi skrip Python utama yang siap dieksekusi menggunakan perintah uv run main.py:

import os
import requests
import asyncio
from urllib.parse import urljoin
from crawl4ai import AsyncWebCrawler
import textstat
import tiktoken
from nltk.corpus import stopwords

# Memeriksa kepatuhan keberadaan llm.txt atau llms.txt
def detect_and_fetch_llm_txt(base_url):
    paths_to_check = ["/llm.txt", "/llms.txt"]
    for path in paths_to_check:
        target_url = urljoin(base_url, path)
        try:
            response = requests.get(target_url, timeout=5)
            if response.status_code == 200 and "text/plain" in response.headers.get("Content-Type", ""):
                print(f"[FOUND] Berkas mesin ditemukan di: {target_url}")
                return response.text, path
        except requests.RequestException:
            continue
    print(f"[NOT FOUND] Platform {base_url} tidak patuh standar llm.txt")
    return None, None

# Deep scraping konten utama halaman web menggunakan Crawl4AI
async def crawl_human_documentation(doc_url):
    async with AsyncWebCrawler() as crawler:
        result = await crawler.arun(url=doc_url)
        if result.success:
            return result.fit_markdown
        return ""

# Menghitung metrik kuantitatif kebahasaan (Manusia vs Mesin)
def calculate_metrics(raw_text):
    words = raw_text.split()
    total_words = len(words)
    if total_words == 0: return None

    # Metrik Keterbacaan Manusia
    fre_score = textstat.flesch_reading_ease(raw_text)
    fk_grade = textstat.flesch_kincaid_grade(raw_text)
    
    # Kepadatan Leksikal (Lexical Density via NLTK)
    stop_words = set(stopwords.words('english'))
    content_words = [w for w in words if w.lower() not in stop_words]
    lexical_density = (len(content_words) / total_words) * 100
    
    # Efisiensi Token AI (Model GPT-4 cl100k_base)
    encoding = tiktoken.get_encoding("cl100k_base")
    token_count = len(encoding.encode(raw_text))
    token_ratio = token_count / total_words

    return {
        "FRE_Score": fre_score,
        "FK_Grade": fk_grade,
        "Lexical_Density": round(lexical_density, 2),
        "Token_Ratio": round(token_ratio, 2)
    }

async def main():
    # Menjalankan pipeline dengan membaca input berkas urls.txt
    with open("urls.txt", "r") as f:
        urls = [line.strip() for line in f if line.strip()]
        
    for base_url in urls:
        llm_text, detected_path = detect_and_fetch_llm_txt(base_url)
        if not llm_text: continue
        
        doc_url = urljoin(base_url, "/docs")
        human_text = await crawl_human_documentation(doc_url)
        
        # Eksekusi kalkulasi metrik kuantitatif
        human_metrics = calculate_metrics(human_text)
        llm_metrics = calculate_metrics(llm_text)
        print(f"Hasil Audit {base_url} -> Sukses Komputasi Data.")

if __name__ == "__main__":
    asyncio.run(main())
